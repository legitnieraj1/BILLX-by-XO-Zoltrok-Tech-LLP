# billX - Premium Point of Sale (POS) & Analytics Platform

**billX** is a cutting-edge, full-stack POS and Inventory Management system designed explicitly for high-end boutique environments (like artisan cafes, bakeries, and retail). It is anchored in the "Digital Concierge" design philosophy—prioritizing depth, glassmorphism, effortless flow, and an authoritative editorial layout.

---

## 🏗 Architecture & Tech Stack

This project is divided into a robust, decoupled frontend and backend to ensure blazing fast tablet performance and reliable data persistence.

### Frontend: The Digital Tablet Interface
- **Framework:** Next.js 14 (App Router)
- **UI Library:** React.js
- **Styling:** Tailwind CSS (with a custom "Surface Layering" minimal border design system)
- **State Management:** Zustand (handling global multi-bill tabs, cart items, search filters)
- **Mobile/Native Wrapper:** Capacitor (allows the Next.js web app to be deployed as a native iOS/Android tablet application)
- **Icons & Typography:** Google Material Symbols (Rounded) & Inter font.

### Backend: The Engine
- **Runtime:** Node.js
- **Framework:** Express.js (REST API)
- **Database ORM:** Prisma v5
- **Database:** SQLite (Currently used for local development flexibility, easily scalable to PostgreSQL for production).

## ⚙️ Core Functionality & Features Built

### 1. Point of Sale (POS) Terminal (`/`)
The heart of the application, designed to be operated at high speed on iPad-sized touch screens.
- **Chrome Tabs System:** Support for managing multiple customer bills simultaneously via a top tab bar. You can open a tab for "Table 1", switch to take a quick takeaway order on "Tab 2", and switch back without losing state.
- **Dynamic Search & Filtering:** A global search bar in the Top Navigation instantly filters down the `MenuGrid` and `QuickPicks` components based on product names. Category pills further refine the grid (e.g., "Beverages", "Bakery").
- **Real-Time Data Fetching:** The POS pulls the live menu database (seeded with the "Cup Culture" menu) from the backend API on load.
- **Smart Cart Panel:** 
  - Allows quick quantity adjustments and item removals.
  - Automatically calculates subtotal, 5% GST tax, and tracks promo discounts.
  - Contextual toggles for "Dine-In", "Takeaway", and "Delivery".
  - One-tap payment mode selection (UPI, Card, Cash).
- **Checkout Flow:** Pressing "Complete Order" captures all cart data, computes the final payload, pushes it to the backend `POST /api/orders` endpoint, flashes a success alert, and clears the active tab.

### 2. Live Analytics Dashboard (`/analytics`)
A C-suite level visualization area updating dynamically based on live POS activity.
- **Real-Time KPIs:** Tracks Total Revenue, Total Orders, Average Order Value, and Net Sales generated by pushing the live Express.js memory arrays through an aggregation loop.
- **Top Selling Items:** Automatically tallies the frequency of purchased SKUs and ranks the top 3 best-sellers complete with their images and revenue generated.
- **Payment Distribution:** A UI representation of the percentage split between UPI, Cash, and Card transactions based on the actual closed orders.
- **Low Stock Warnings:** Communicates with the Prisma database to extract products where `stock <= 10` and pushes them dynamically to the "Inventory Insights" table, urging the merchant to reorder.
- **Timeframe Toggles:** UI state implemented for swapping between Today, Week, Month, and Custom reports. 
- **Print SDK:** A "Download Report (PDF)" button wired into the browser's native print engine for physical accounting.

### 3. Inventory Management Admin (`/inventory`)
A fast, inline data-grid designed for warehouse and stock managers.
- **Live Database Link:** Fetches all Prisma products.
- **Inline Pricing:** Prices are rendered in actionable inputs allowing for quick text-box modifications (wired to hit the `PATCH` API).
- **Stock Toggles:** 1-click pills to visibly swap an item from "In Stock" to "Out of Stock".
- **Visual Grid:** Beautifully renders the database images alongside SKUs and categorized tags so managers can identify visual items quickly.
- **Scrollable Layout:** A sticky-header table ensures that even with 80+ items, the action buttons (+ Add Product) and headers remain visible as you scroll the inventory list.

## 🗄 Database Schema (Prisma)

The application utilizes a relational scheme to maintain data integrity.

- **`Category` Model:** Represents menu families (e.g., "Espresso", "Cold Brew").
- **`Product` Model:** Houses the specific items (`price`, `stock`, `sku`, `lowStockAlert`, `isAvailable`). Belongs to a single Category.
- **`StockLog` Model:** An auditable ledger to track who added or removed stock, by what quantity, and when (e.g., "Delivery received 50 units" or "Spoilage removed 2 units").

The database was successfully seeded using a massive `seed.js` script containing the fully extracted "Cup Culture" physical menu, populating the POS with over 30+ highly detailed artisanal cafe items, distinct tags, and base stock levels.

## 🚀 How to Run the Stack Locally

You must run *both* the backend and the frontend concurrently.

### 1. Start the Backend API
The Node API and Prisma Database run on `http://localhost:4000`.

```bash
cd backend
npm install
npx prisma generate
npx prisma db push
node seed.js # (If you need to reset the Cup Culture menu)
npm run dev
```

### 2. Start the Frontend Next.js Application
The Frontend runs on `http://localhost:3000`.

```bash
cd billx-pos
npm install
npm run dev
```

### 3. View the Application
Open [http://localhost:3000](http://localhost:3000) in your web browser. 
*(For the best experience, open Chrome DevTools and set the viewport to an iPad Pro to see the intended layout scale).*
